Stacking context

Overlays contained in elements that create a new stacking context are trapped within it.
E.g. Click on trapped and notice how the overlay renders behind the backdrop and scrolls together with the rest of the content.

iron-overlay-template solves this issue while still providing a reference to the overlay at the spot where it is created.

Styles can be added inside the template.

iron-dropdown

iron-dropdown positions itself according to a positionTarget element (usually the parent node).
The parent node can implement iron-overlay-factory-behavior to overcome stacking context issues, and assign itself as the positionTarget.

iron-dropdown in shadowDOM

Elements that need an iron-dropdown in their shadowDOM (e.g. paper-menu-button) usually distribute partial content in it. Keeping the same markup would require to detach & attach either the dropdown or its distributed content.

To avoid this, use iron-overlay-content-template to stamp overlay content; x-menu-button (which implements iron-overlay-host-behavior) will ensure the content is added to the dropdown.

Furthermore, the element author can style the overlay, and define where the content should be distributed in the overlay using the overlay-content-parent attribute.
